# 版本更新紀錄

## v1.0
- **原始需求 Prompt**: 優化圖表預設縮放範圍：每次進 OvenDetail 圖表時皆從最右側的最新數值開始看（類似鎖定跟隨模式 Auto-Tail）。若加入一鍵「Return to Live」按鈕...
- **實作細節**:
  - **自動跟隨模式 (Auto-Tail)**：在 `OvenDetail` 實作了 `isAutoTail` 狀態。當進入即時的一小時視角時，圖表會自動鎖定並跟隨顯示最新的三十筆數據；若遇到視窗更動皆會智能計算靠右對齊。
  - **介入偵測與解除鎖定**：當使用者對圖表進行滾輪縮放 (Wheel Zoom) 或滑鼠拖拉 (Pan) 時，系統會自動解開 Auto-Tail 鎖定，讓工程師能自由回溯檢視過去三天或任何時間點的舊數據。
  - **一鍵返回即時 (Return to Live)**：在圖表控制列新增了一顆極具科技感、帶有呼吸燈動畫的「Return to Live」按鈕。當脫離自動跟隨或在閱覽歷史日期時，點擊此按鈕將瞬間將所有時間參數（年、月、日、時）切換回當下，並重新鎖扣 `isAutoTail` 狀態，完美無縫銜接最新的 5 秒推移資料流。

## v0.9
- **原始需求 Prompt**: 為什麼需要用到GEMINI API KEY,移除相關API KEY的使用
- **實作細節**:
  - **環境清理**：移除了專案預設範本中綁定的 `GEMINI_API_KEY` 環境變數。因為目前此烤箱監控系統完全是由本地邏輯（React + Tailwind + 動態模擬資料）來實現工廠端 OEE 分析與圖表推移，並無依賴任何大型語言模型（LLM）等生成式 AI 輔助流程，確實不需要該 API Key 造成不必要的隱私考量或載入累贅。
  - 同步移除了 `.env.example` 宣告及 `vite.config.ts` 中的底層 `process.env` 映射。

## v0.8
- **原始需求 Prompt**: 1.強化圖表平滑化 2.結合警報推播的 Service Worker (離線支援) 3.增加資料的群組或工區分類 (Zones)
- **實作細節**:
  - **圖表完全平滑化**：在 `OvenDetail` 中，將歷史溫度圖表中 `Line` 與 `Area` 組件的 `type` 屬性統一由預設或 `monotone` 轉換為 `natural`。現在圖表連線變為極度平滑的貝茲曲線，配合心電圖推移動畫大幅提升了科技感與絲滑度。
  - **PWA / Service Worker (離線支援)**：正式安裝並引入 `vite-plugin-pwa` 套件，在系統層級註冊了 Service Worker (`sw.js`) 並配置基礎 Manifest。這賦予了系統在背景運行時的能力藍圖，且在離線與網路不穩時能透過快取保障部分資料的安全與介面響應。
  - **廠房分區群組 (Zones)**：
    - 將烤箱資料結構 (`Oven` type) 新增了 `zone` 分區屬性，於模擬資料中平均將 18 台設備分派至「A區、B區、C區」。
    - 在 Dashboard 總覽面版的右上角新增了分頁頁籤 (Tabs)，可一鍵從「全部區域」快速過濾或專注即時監控特定工區的設備。
    - 卡片標題旁同步加註各機器的區域名稱標籤，讓大量管理更具直覺性。

## v0.7
- **原始需求 Prompt**: 曲線圖頁面不要超過螢幕邊界造成有scroll
- **實作細節**:
  - 修復版面外溢：將整體 `App` 的基礎結構從 `min-h-screen` 設為強制固定 `h-screen`。
  - 改寫圖表佔比：調整 `OvenDetail` 內部為 `overflow-hidden`，並將外層網格 Grid 改為 Flex 佈局。賦予 Chart 跟 Sensor 列表各自專屬的捲動狀態 (overflow-auto)，確保當螢幕較窄時，主體能順利包夾而不會撐開全版產生垂直捲軸。
  - 將圖表的最小高度彈性降低到 200px 確保小螢幕也能正常滿版顯示，解決了圖表面板強制拉出全域捲軸的瑕疵。

## v0.6
- **原始需求 Prompt**: 1.即時同步最新資料推移點(心電圖) 2.加入設備稼動率分析(OEE) 3.行動裝置響應式優化(圖表全螢幕按鈕) 4.加入系統聲音音效輔助 5.引入自動刷新計時器 UI 6.側邊欄可以縮放，縮成只有圖示
- **實作細節**:
  - **心電圖 (即時推移圖表)**：全面重構 Store，在 `isCurrentTime` 的時檢視邏輯下，圖表能自動監聽更新週期 (5秒) 並拉回最新實時感測資料，呈現如心電圖般逐漸向左推進的動畫感。
  - **OEE (稼動率 / Overall Equipment Effectiveness)**：在 Store 資料結構與 Mock 中新增設備綜合效率屬性，該屬性會隨著溫度狀況 (Normal 提升，Critical 下降) 動態微調，並顯示於總覽卡片右下角呈現。
  - **音效提示**：使用瀏覽器內建 Web Audio API (`AudioContext`) 實作了單純無依賴的清脆長音 BEEP 提示，只要出現 Critical 通知 (如測試警報、自動生成的高溫)，皆會伴隨強烈聲響。
  - **自動刷新計時器 UI**：於總覽 (Dashboard) 頁面右上角新設置了倒數計時元件 (5~0秒)，清晰傳達系統每5秒自動更新最新數據的頻率。
  - **全螢幕圖表、RWD 優化**：重寫 OvenDetail (詳細面版) 的圖表格線容器，右上角加入全螢幕/縮小圖示，並實裝雙擊圖表主體即可橫跨螢幕全版面展示的能力，十分適合行動裝置。
  - **可收摺側邊欄**：左側選單的 Sidebar 加入了折疊能力 (`isSidebarOpen`)，可展開為含文字選單，或點選漢堡選單縮編至僅剩圖示與縮寫版本號，為小螢幕保留更多主內容使用空間。

## v0.1
- **原始需求 Prompt**: 我要做一個工廠內烤箱溫度監控網站，需要記錄歷史溫度，並能顯示即時溫度。有18台烤箱，每台烤箱內有3到5個溫度計，點選溫度計時可以看到詳細的溫度曲線圖表及能看到該烤箱所有的溫度計。曲線圖表要有年、月、日、小時、分鐘的篩選功能，另外能設定溫度異常時的警報通知。警報時在畫面上能有紅色閃爍的標示，並有推播通知到管理者手機。補充：警報類型也能在系統中設定、警報記錄需要包含觸發時間、烤箱編號、溫度計編號、異常溫度值。
- **實作細節**:
  - 初始化專案，安裝 `recharts`, `date-fns`, `clsx`, `tailwind-merge`, `lucide-react` 等套件。
  - 建立基礎架構與路由切換 (Dashboard, Oven Detail, Alarms, Settings)。
  - 實作 Mock Data 產生器，模擬 18 台烤箱與感測器即時資料。
  - 採用 Technical Dashboard 風格設計 UI。
